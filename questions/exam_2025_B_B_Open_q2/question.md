## Question
(2 נקודות) איזה עיקרון בעיצוב חבילות מופר בתרשים שלפניך?
(5 נקודות) כיצד ניתן לתקן זאת ללא העברה של מחלקות מחבילה לחבילה? שרטט תרשים חדש. אין צורך לשרטט מחדש חלקים בתרשים שלא השתנו. הסבר כיצד השינוי פותר את ההפרה.
![Image](image_4_1.png)

## Answer
**א. העיקרון המופר:**
העיקרון המופר הוא **Acyclic Dependencies Principle (ADP) - עקרון התלות הלא-מחזורית**. עיקרון זה קובע כי גרף התלויות בין חבילות במערכת צריך להיות חסר מחזורים (DAG - Directed Acyclic Graph). בתרשים המקורי, קיימת תלות מחזורית בין החבילות P1, P2 ו-P3:
P1 -> P2 (לדוגמה: C תלוי ב-H, A תלוי ב-H)
P2 -> P3 (לדוגמה: H תלוי ב-D, J תלוי ב-E)
P3 -> P1 (לדוגמה: D תלוי ב-A, F תלוי ב-C)

**ב. תיקון ההפרה:**
כדי לתקן את ההפרה מבלי להעביר מחלקות בין חבילות, נשתמש ב**Dependency Inversion Principle (DIP)**. נהפוך את התלות מ-P3 ל-P1 על ידי יצירת ממשק בחבילה P3, ש-P1 תממש. כך, P3 תהיה תלויה בממשק שהיא עצמה מגדירה, ו-P1 תהיה תלויה בממשק זה (באמצעות מימוש), ובכך נשבור את המחזוריות.

**שלבי התיקון:**
1.  **זיהוי התלויות המחזוריות:** התלויות מ-P3 ל-P1 הן: D->A, E->B, F->C, G->A.
2.  **היפוך תלות:** עבור כל מחלקה ב-P1 שבה תלויה מחלקה מ-P3, ניצור ממשק בחבילה P3. לדוגמה:
    *   ניצור ממשק `IA` ב-P3. מחלקה `A` ב-P1 תממש את `IA`.
    *   ניצור ממשק `IB` ב-P3. מחלקה `B` ב-P1 תממש את `IB`.
    *   ניצור ממשק `IC` ב-P3. מחלקה `C` ב-P1 תממש את `IC`.
3.  **עדכון התלויות ב-P3:** המחלקות D, E, F, G ב-P3 יתלו עתה בממשקים `IA`, `IB`, `IC` במקום במחלקות `A`, `B`, `C` ישירות.

**תרשים מתוקן (רק החלקים שהשתנו):**

**חבילה P1:**
*   `A` (מממש את `IA`)
*   `B` (מממש את `IB`)
*   `C` (מממש את `IC`)

**חבילה P3:**
*   `<<interface>> IA`
*   `<<interface>> IB`
*   `<<interface>> IC`
*   `D` (תלוי ב-`IA`)
*   `E` (תלוי ב-`IB`)
*   `F` (תלוי ב-`IC`)
*   `G` (תלוי ב-`IA`)

**הסבר כיצד השינוי פותר את ההפרה:**
לאחר השינוי, התלויות הן:
*   P1 -> P2 (ללא שינוי)
*   P2 -> P3 (ללא שינוי)
*   P3 -> P3 (D, E, F, G תלויים בממשקים IA, IB, IC שנמצאים ב-P3)
*   P1 מממש את הממשקים ב-P3 (P1 תלוי ב-P3 לצורך מימוש, אך זו תלות בכיוון ההפוך מהתלות המקורית ששברנו).

המחזוריות P3 -> P1 נשברה. כעת, P3 תלויה רק בעצמה (בממשקים שהיא מגדירה), ו-P1 תלויה ב-P3 דרך מימוש הממשקים. זה יוצר גרף תלויות חד-כיווני (P1 -> P2 -> P3), ובכך עומד בעיקרון ADP. היתרון הוא ששינויים ב-P1 לא יחייבו שינויים ב-P3 (אלא אם כן הממשקים ב-P3 משתנים), ו-P3 הופכת לחבילה יציבה יותר.
